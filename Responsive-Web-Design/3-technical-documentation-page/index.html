Introduction

Bitcoin transactions are short text files that contain information necessary to validate a double entry on the bitcoin blockchain. Each transaction contains a credit that demonstrates ownership and validation of the spend of an input unspent transaction output (UTXO). The transaction debits the spent bitcoin to a locking script that belongs to the destination of the funds.

Transactions are propagated through the bitcoin network to each node for validation against the existing data of the blockchain stored on each node. Mining nodes pick up valid transactions and build blocks to permanently lock in the transactions on the blockchain. The blockchain can be considered an append-only ledger.
